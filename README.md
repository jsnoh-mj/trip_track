# Trailfilm by MJ — 사용 매뉴얼

GPX 경로와 사진만 있으면, 3D 지형 위를 날아다니는 시네마틱 여정 영상을 만들어주는 개인용 웹 앱입니다.
서버·설치 없이 HTML 파일 하나로 동작하며, Relive 같은 상용 서비스의 "자동 스토리텔링" 컨셉을 개인이 직접 세밀하게 연출할 수 있도록 만든 도구입니다.

## 1. 컨셉과 목적

### 왜 만들었나
등산·자전거·여행 앱으로 GPX 경로를 기록하고, 같은 시간에 사진을 찍었다면 — 그 둘을 자동으로 짝지어 "경로를 따라가며 그 순간마다 사진이 떠오르는" 영상을 만들 수 있습니다. Relive 같은 서비스가 이런 걸 자동으로 해주지만, 세부 연출(카메라 각도, 카드에 적을 문구, 사진 없는 지점에 코멘트 남기기 등)을 직접 만지기는 어렵습니다. Trailfilm은 그 자동화는 그대로 가져오되, **모든 디테일을 사람이 직접 다듬을 수 있게** 만든 도구입니다.

### License 및 책임
주의! 이 앱은 개인 사용은 누구나 허락하나, 만약의 어떠한 SW에 대한 보증이나 아이디어에 대한 문제는 이 공유하는 사람이 책임지지 않습니다. 이것을 사용하는 사람이 개인 사용 및 SW에 대해서 특이점이 있는지는 AI등에 의뢰해서 확인후 사용하시기 바랍니다. 그리고, 이 tool 의 내용을 그대로 가져가서 상업적인 SW개발에 사용하는 것은 금지 합니다. 개인적으로 이것으로 AI를 이용해서 개선하여 사용하는 것은 자유 입니다. 이 SW는 순수하게 Claude AI를 통해서 만들었습니다. 기타 어떠한 상용 SW의 도움을 받지 않았습니다.

### 기본 동작 원리
1. GPX 파일에서 위치·시간·고도 정보를 읽습니다.
2. 사진의 EXIF 촬영시각(또는 파일명)을 GPX의 시간과 대조해서, 사진이 찍힌 정확한 위치를 경로 위에 자동으로 배치합니다.
3. 카메라가 경로를 따라 3D 지형 위를 날아다니다가, 사진/코멘트/영상이 있는 지점에서 잠시 멈춰 카드를 띄웁니다.
4. 완성된 화면을 OBS 같은 화면 녹화 프로그램으로 캡처하면 바로 유튜브·블로그용 영상이 됩니다.

### 이 앱의 성격
- **개인/지인용 취미 도구**입니다. 상용 서비스가 아니고, 서버도 없고, 인터넷에 뭔가를 업로드하지도 않습니다. 모든 처리는 내 컴퓨터의 브라우저 안에서만 일어납니다.
- 파일 하나(`trailfilm.html`)로 동작하며, 인터넷 연결은 필요합니다(지도 타일, 폰트, 라이브러리를 온라인에서 불러오기 때문).

## 2. 주요 기능

### 2.1 경로 · 사진 매칭
- GPX의 `<trkpt>` 시간·좌표·고도를 읽어 누적거리 기준으로 경로를 만듭니다.
- 사진은 EXIF의 `DateTimeOriginal`(촬영시각)을 GPX 시간과 대조해 정확한 위치로 보간합니다. 사진 자체에 GPS 태그가 있으면 그 좌표를 그대로 씁니다.
- **파일명 자동 인식**: 사진 파일명이 `YYYYMMDD_HHMMSS_설명.jpg` 형식이면(앞에 `IMG_`, `GOPR_` 같은 접두어가 붙어도 인식) `_설명` 부분을 코멘트 기본 텍스트로 자동으로 씁니다. 예) `20260612_134402_장군봉.jpg` → "장군봉"
- 카드에는 거리·고도 정보가 `- 1.2 Km / H 1692m` 형식으로 자동으로 붙습니다.
- EXIF 시간을 못 읽은 사진(HEIC 등)은 자동으로 4초 안에 넘어가고, 선택 순서 기준으로 경로에 고르게 배치되며 "위치 추정" 표시가 붙습니다.

### 2.2 3D 지도 & 카메라 연출
- **지도 공급자 2종류**를 첫 화면에서 선택할 수 있습니다.
  - **MapLibre (기본)**: 무료, API 키 불필요. 다만 지역에 따라 식생 표현이 부실할 수 있습니다.
  - **MapTiler (지형개선 모드)**: API 키 필요(https://cloud.maptiler.com/ 구글계정 연동후 개인 API key를 받아서 입력, 무료 티어 가능하나 API 수량 제한 있음). "Outdoor"(일러스트풍) 또는 **"Satellite Hybrid"(실사 위성사진+지명, 항상 진짜 초록)** 중 선택 가능. 산이 예쁘게 나오길 원하면 Satellite Hybrid를 추천합니다.
- 카메라는 경로를 따라가며:
  - 진행 방향을 부드럽게(급커브에서도 홱 돌지 않게) 따라갑니다.
  - 완만한 좌우 사이드뷰 변주로 계속 같은 각도로 고정되지 않습니다.
  - 협곡·계곡(고도가 급하게 낮아지는 구간)에서는 자동으로 살짝 가까워지는 다이나믹한 움직임을 줍니다.
  - 카드가 떠 있는 동안은 완전히 정지해서 시청에 방해되지 않습니다.
- 지나온 경로는 **현재 위치 쪽이 가장 밝게 빛나는 그라데이션 선**으로 그려져서, 왔던 길을 되돌아가도 진행 방향을 헷갈리지 않습니다.
- 이동 아이콘(사람🚶/자전거🚴/차🚗, 첫 화면에서 선택)은 경로선과 다른 색(청록)으로 표시되어 항상 눈에 잘 띕니다.

### 2.3 카드 (사진 / 코멘트 / 영상)
- **사진 카드**: 크림색 카드에 사진 + 자동/직접 작성한 코멘트가 뜹니다.
- **코멘트(글자만)**: 카드 없이 지도 위에 큰 글씨만 풍선처럼 떠올랐다 사라집니다. 지명 표시("당골고개", "데크시작") 등에 적합합니다.
- **영상 카드**: 로컬 파일 또는 URL로 영상을 넣을 수 있고, 카드는 영상 재생시간 + 앞뒤 0.3초만큼 떠 있습니다. (액션캠에서 뽑은 스틸컷처럼 EXIF 없는 사진도 코멘트/영상 카드로 원하는 지점에 직접 추가 가능)
- 모든 카드는 다음을 개별 조정할 수 있습니다: **텍스트, 글자색(흰/검/빨/초/파), 표시 시간(0.5초 단위), 카메라 시점(좌우 각도·거리·피치)**.
- 카드는 이동 아이콘 위에 바로 겹치지 않고, 살짝 위에 풍선처럼 떠서 선으로 연결됩니다.

### 2.4 인트로(타이틀) 카드
- 재생 시작마다 여정 제목·소개가 자동으로 뜹니다. 대표 사진(전체 경로 스냅샷 등)을 넣으면 사진 있는/없는 레이아웃이 각각 예쁘게 자동으로 바뀝니다.
- dock의 "🎬 인트로" 버튼으로 켜고 끌 수 있습니다(OBS로 이어 찍을 때 반복해서 안 보고 싶으면 끄면 됩니다).

### 2.5 디렉터 패널 (편집 기능)
- 화면 오른쪽 상단 "편집창 열기"로 열고 닫습니다. 지도 위에 겹쳐 뜨는 사이드 패널이라 플레이어 화면 구조는 그대로입니다.
- **재생 중에는 편집이 잠깁니다.** 일시정지(❚❚) 상태에서만 수정할 수 있습니다. 재생 중 자동으로 뜬 카드에서 일시정지를 누르면, 그 카드는 자동으로 안 닫히고 편집할 시간을 충분히 줍니다.
- 카드 목록에서 원하는 카드를 클릭하면 그 지점으로 바로 이동해서 편집할 수 있습니다.
- 여정 제목/소개/대표사진/인트로 시간도 언제든 다시 수정 가능합니다.
- **자동 저장**: 텍스트·색상·타이밍·카메라 보정값은 브라우저에 자동 저장되어, 같은 GPX+사진 조합을 다시 열면 복원됩니다(파일명 기준 매칭). 단, EXIF 없이 새로 추가한 사진/영상 원본은 용량 문제로 복원이 안 될 수 있어 "재첨부 필요" 표시가 뜰 수 있습니다. "🗑 저장된 편집 초기화" 버튼으로 언제든 리셋할 수 있습니다.

## 3. 사용 방법

### 3.1 준비물
- **GPX 파일** 1개 (등산/자전거 앱에서 내보낸 경로 기록).
- **사진 여러 장** — 되도록 아래 파일명 규칙을 지키면 코멘트가 자동으로 채워집니다.
  ```
  YYYYMMDD_HHMMSS_설명.jpg
  예) 20260612_134402_장군봉.jpg
  ```
- (선택) 인트로에 쓸 **대표 사진** 1장 — 전체 경로가 보이는 지도 스냅샷 등.
- (선택) **MapTiler API 키** — [cloud.maptiler.com](https://cloud.maptiler.com/account/keys/)에서 무료로 발급 (신용카드 불필요).

### 3.2 앱 열기
1. `trailfilm.html` 파일을 파일 탐색기에서 **더블클릭**하면 기본 브라우저로 바로 열립니다. 서버가 없어도 정상 동작합니다.
   - 인터넷 연결은 필요합니다(지도/폰트/라이브러리를 온라인에서 불러오기 때문).
   - 혹시 안 열리거나 이상하면: 탭을 완전히 닫고 다시 열거나, `Ctrl+Shift+R`(강력 새로고침), 또는 시크릿 창에서 열어보세요.

### 3.3 여정 만들기 (첫 화면)
1. **여정 제목 / 여정 소개**를 입력합니다 (인트로 카드에 표시됩니다).
2. **대표 사진**을 선택합니다 (선택 사항).
3. **인트로 표시 시간**을 조정합니다 (2~15초).
4. **GPX 경로 파일**을 선택합니다.
5. **사진들**을 선택합니다 (여러 장 한 번에 가능).
6. **이동 수단 아이콘**을 고릅니다 (등산·자전거·차량).
7. **지도 공급자**를 고릅니다.
   - 그냥 무료로 쓰려면 MapLibre(기본) 그대로 두면 됩니다.
   - 산을 더 예쁘게(진짜 초록으로) 보고 싶으면 MapTiler 선택 → API 키 입력 → **Satellite Hybrid** 스타일 추천.
8. **"여정 시작하기"** 버튼을 누릅니다.
   - GPX 분석 → 사진 EXIF 읽기(진행률 표시) 순으로 처리된 뒤 자동으로 지도 화면으로 넘어갑니다.

### 3.4 재생 화면 조작
화면 하단 컨트롤 바(dock):
| 버튼 | 기능 |
|---|---|
| ▶ / ❚❚ | 재생 / 일시정지 |
| 처음부터 | 인트로부터 다시 시작 |
| 🎬 인트로 | 인트로 카드 표시 on/off |
| 0.5× ~ 4× | 재생 속도 |
| 하단 능선 그래프 | 실제 고도 프로파일이자 타임라인 — 드래그해서 원하는 지점으로 이동, 점 클릭 시 해당 카드로 바로 이동 |

### 3.5 편집하기 (디렉터 패널)
1. 화면 우상단 **"✎ 편집창 열기"** 클릭 (재생 중이면 먼저 ❚❚로 정지하세요 — 재생 중엔 편집이 잠깁니다).
2. **여정 정보**: 제목/소개/대표사진/인트로 시간을 다시 수정할 수 있습니다.
3. **카드 추가**: 코멘트/사진/영상 중 타입을 고르고, 텍스트·색상·표시시간(또는 영상 파일/URL)을 입력한 뒤 **"이 지점에 카드 추가"**를 누르면 현재 재생 위치(스크러버 위치)에 카드가 생깁니다.
   - 영상 카드는 표시시간이 영상 길이 기준으로 자동 계산됩니다.
   - 사진 카드는 EXIF 없는 사진(액션캠 스틸컷 등)도 자유롭게 추가할 수 있습니다.
4. **현재 카드 편집**: 카드 목록에서 카드를 클릭하거나, 재생 중 자동으로 뜬 카드에서 일시정지하면 이 섹션이 활성화됩니다. 텍스트(카드/화면에서 직접 클릭해 타이핑), 표시시간(0.5초 단위), 글자색, 카메라 시점(좌우 각도·거리·피치)을 조정할 수 있습니다.
5. **카드 목록**: 전체 카드를 순서대로 보여주며, 클릭하면 바로 그 지점으로 이동, ✕로 삭제할 수 있습니다.
6. 다 고쳤으면 카드는 **✕ 버튼** 또는 **▶ 재생**을 누르면 닫히고 재생이 이어집니다.
7. 수정 내용은 자동 저장됩니다. 다시 새로 시작하고 싶으면 **"🗑 저장된 편집 초기화"**를 누르세요.

### 3.6 영상으로 뽑기 (OBS 등으로 녹화)
1. 원하는 대로 다 편집했으면 편집창을 닫습니다 ("✕ 편집창 닫기").
2. "처음부터"를 눌러 인트로부터 재생을 시작합니다.
3. OBS Studio 등으로 브라우저 창(또는 지정 영역)을 화면 녹화합니다.
4. 녹화된 화면을 다빈치 리졸브 등 영상 편집 프로그램에서 배경음악·자막 등을 더해 완성합니다.

## 4. 알아두면 좋은 점 / 제약사항

- **로컬 저장(localStorage) 용량 한계**: 텍스트·타이밍·색상·카메라 보정값은 안정적으로 저장되지만, 새로 추가한 큰 사진/영상 원본은 브라우저 저장 용량(보통 5~10MB) 때문에 새로고침 후 다시 첨부해야 할 수 있습니다. 카드 목록에 "⚠ 재첨부 필요"로 표시됩니다.
- **API 키 노출**: 서버가 없는 구조라 MapTiler API 키가 브라우저 개발자도구에 그대로 보입니다. 즉, API 등을 실행하신 directory 전체를 git이나 타인에 노출하지 마세요. 그리고, html을 수정하여 직접 API key를 html에 하드코딩 하지 마세요.
- **지형 가림(occlusion) 판정**: 카메라가 능선 뒤에서 이동 아이콘을 완전히 가리는지까지는 정밀하게 계산하지 않고, 피치 범위를 보수적으로 유지하는 근사치로 처리합니다.
- **HEIC 파일**: 브라우저가 HEIC의 EXIF/미리보기 처리를 잘 못하는 경우가 있어, JPEG로 변환해서 쓰는 걸 권장합니다.
- **MapTiler 사용량**: 카메라가 경로 전체를 계속 이동하며 타일을 요청하는 구조라, 일반적인 "지도 한 번 보기"보다 타일 요청이 많이 발생합니다. 무료 한도 내에서 여러 번 테스트해보고 계정 대시보드에서 사용량을 확인해보세요.
